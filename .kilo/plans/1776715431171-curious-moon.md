# Plan: Redesign `wtg-sheet` to Premium Grid + Animated Detail Panel

## Overview
Transform the `WeeklyTopGrid` (`wtg-sheet`) from a basic image+name grid into a high-conversion, modern interface. Grid cells display **only the sold quantity**. On click, the grid slides left with a smooth animation and a detail panel appears on the right showing the **product name, product image, and quantity sold**.

## Key Design Decisions
- **Close behavior:** Detail panel has an X button.
- **Current-week highlight:** Blue border remains on the current week's cell.
- **Empty/future cells:** Not clickable (no cursor pointer, no hover effect).
- **Animation strategy:** Pure CSS transitions (no new dependencies). A parent flex container shifts from `justify-content: center` to `justify-content: flex-start`, and the detail panel transitions `opacity` + `transform` + `max-width`.
- **Responsive behavior:** On viewports < 600px, the detail panel becomes a fixed overlay/sheet from the bottom instead of a side panel, to preserve usability.
- **Theme:** Retain the existing dark Notion-style theme (`#191919` background) for visual consistency with the current `WeeklyTopGrid`.

## Step-by-Step Implementation

### Step 1: Refactor `src/components/WeeklyTopGrid/index.jsx`
1. **Add state:**
   - `selectedWeek` — holds the week object of the clicked cell, or `null`.
   - `isAnimating` — brief flag to coordinate exit animation if needed.

2. **Update `Cell` component:**
   - Receive `onClick` and `isSelected` props.
   - Render ONLY the quantity number (`topProduct.qty`) in the center of the cell.
   - Keep the current-week blue border (`wtg-cell--current`).
   - Keep `wtg-cell--future` and `wtg-cell--missing` styles.
   - Add hover state for clickable cells (non-empty, non-future).
   - Add `cursor: pointer` only when clickable.
   - Apply a selected state style (subtle glow or scale) when `isSelected`.

3. **Create `DetailPanel` component (inside the same file or inline):**
   - Accepts `week` object and `onClose` callback.
   - Displays:
     - Large product image (`topProduct.imageUrl`)
     - Product name (`topProduct.name`)
     - Quantity sold (`topProduct.qty`)
     - Week label (e.g., "Semana 2 — Enero") for context
   - Includes a close button (X) using `lucide-react`'s `X` icon.
   - Animates in with `opacity` and `translateX`.

4. **Restructure the layout:**
   - Wrap `wtg-sheet` and the new `DetailPanel` in a flex row container (`wtg-stage`).
   - Default: `justify-content: center` → grid is centered.
   - When `selectedWeek` is set: add a `wtg-stage--active` class → `justify-content: flex-start` and the grid gets a left-slide transform.
   - The detail panel renders conditionally when `selectedWeek` is truthy.

5. **Preserve existing logic:**
   - Keep `HighlightPill` components and `wtg-highlights` carousel.
   - Keep `wtg-caption`, loading spinner, and footnote.
   - Keep `sheet` memoization and `colCount` calculation.

### Step 2: Refactor `src/components/WeeklyTopGrid/styles.css`
1. **Grid cell redesign:**
   - Remove image and footer from `.wtg-cell`.
   - Center the quantity number with `display: grid; place-items: center`.
   - Increase font size for the quantity (e.g., `20px` bold) for visual impact.
   - Keep `aspect-ratio: 1 / 1` (or slightly adjusted) for a clean square look.
   - Add subtle hover effect for clickable cells: `transform: scale(1.05)` + brighter border.
   - Add active/selected state: ring or glow around the selected cell.

2. **Stage layout:**
   - `.wtg-stage`: `display: flex; align-items: flex-start; gap: 16px; width: 100%; transition: justify-content 0.4s ease`.
   - `.wtg-sheet`: `transition: transform 0.4s cubic-bezier(0.34, 1.2, 0.64, 1)`.
   - When `.wtg-stage--active` is present: `.wtg-sheet` gets `transform: translateX(0)` (anchored left) and the stage switches to `justify-content: flex-start`.
   - Actually, a better approach: keep the stage centered by default. When active, the sheet shrinks or moves left, and the panel slides in from the right. Use `flex` layout where panel has `max-width: 0 → 280px` transition.
   - Refinement: `.wtg-stage` is `display: flex; width: 100%; justify-content: center; transition: all 0.4s ease`. When active: `justify-content: center` still, but `.wtg-sheet` scales down slightly and `.wtg-panel` gains `max-width` and `opacity`. **Better UX:** On desktop, grid moves to the left third and panel takes the right third. This can be done with a grid layout or flex with `margin-right: auto` when inactive.
   - Final CSS strategy:
     - `.wtg-stage`: `display: flex; width: 100%; gap: 0; justify-content: center; transition: gap 0.4s ease`.
     - `.wtg-sheet`: `flex-shrink: 0; transition: transform 0.4s cubic-bezier(0.34, 1.2, 0.64, 1)`.
     - `.wtg-panel`: `max-width: 0; opacity: 0; overflow: hidden; transition: max-width 0.4s ease, opacity 0.3s ease 0.1s`.
     - `.wtg-stage--active`: `gap: 24px; justify-content: center` (or flex-start with padding). `.wtg-panel` gets `max-width: 280px; opacity: 1`.

3. **Detail Panel styling:**
   - Dark card matching the grid theme (`#242424` bg).
   - Rounded corners (`16px`), subtle shadow.
   - Large product image area (`object-fit: cover`, `border-radius: 12px`).
   - Typography: product name in `16px` semibold white, quantity in `32px` bold accent color (`#529cff`).
   - Close button: top-right, `24px` circle, hover opacity change.
   - Min-width when open: `260px`.

4. **Responsive (mobile):**
   - Below `600px`: when active, the detail panel becomes a fixed bottom sheet (`position: fixed; bottom: 0; left: 0; right: 0; z-index: 100`) that slides up. The grid does not slide left; instead it dims slightly.
   - Add a backdrop overlay on mobile when panel is open.

5. **Cleanup:**
   - Remove old `.wtg-cell-img`, `.wtg-cell-footer`, `.wtg-cell-name` styles or repurpose them for the detail panel.
